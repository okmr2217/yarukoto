# バージョン更新プロセスガイド

> Claude 参照用ドキュメント。新バージョンを追加するときに参照する。

---

## 基本方針

- バージョン番号の**正本は `package.json` の `version`**（他ファイルで重複管理しない）
- 変更履歴は `CHANGELOG.md` に記録（ユーザー視点で書く）
- 新バージョン追加時は必ず Git 履歴・差分を確認してから書く（推測禁止）
- `handoff.md` のヘッダーにあるバージョン番号も合わせて更新する

---

## バージョン番号の規則（Semantic Versioning）

```
MAJOR.MINOR.PATCH

0.1.0  → 0.1.1  : バグ修正・軽微な改善（PATCH）
0.1.0  → 0.2.0  : 新機能追加（MINOR）
0.x.x  → 1.0.0  : 初回リリース・大規模変更（MAJOR）
```

---

## 新バージョン追加手順

1. **Git 履歴・差分を確認する**（推測で書かない）

   ```bash
   git log --oneline -20
   git diff HEAD~N HEAD
   ```

2. **`package.json` のバージョンを更新する**

   ```json
   { "version": "0.2.0" }
   ```

3. **`CHANGELOG.md` に追記する**（ユーザー視点で、技術詳細は書かない）

   ```md
   ## 0.2.0 - YYYY-MM-DD

   - 新機能の説明（ユーザーにとって何が変わったか）
   - バグ修正の説明
   ```

4. **`docs/handoff.md` のヘッダーを更新する**

   ```md
   > バージョン: 0.2.0
   ```

5. **コミットする**

   ```
   chore: v0.2.0 にバージョンアップ
   ```

---

## CHANGELOG の書き方

### 良い例（ユーザー視点）

```md
## 0.2.0 - 2026-04-01

- タスクのドラッグ＆ドロップで並び替えができるようになりました
- カレンダー画面から日付別タスクに直接移動できるようになりました
- タスク詳細画面でメモ内の URL がリンクになりました
```

### 悪い例（技術詳細）

```md
## 0.2.0 - 2026-04-01

- @dnd-kit を導入して displayOrder Float 方式で並び替えを実装
- DateCell クリックで /dates/[date] にルーティング
- URL 正規表現でリンク化処理を追加
```
